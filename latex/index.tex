 There are two ways to run {\ttfamily huron}\+: using the prebuilt Docker image or building from source.

\DoxyHorRuler{0}
\hypertarget{index_autotoc_md1}{}\doxysection{1. Running with Docker\+:}\label{index_autotoc_md1}
\hypertarget{index_autotoc_md2}{}\doxysubsection{Prerequisites\+:}\label{index_autotoc_md2}

\begin{DoxyItemize}
\item Docker Engine/\+Desktop
\item Computer with amd64 or arm64 chip
\end{DoxyItemize}\hypertarget{index_autotoc_md3}{}\doxysubsection{Steps\+:}\label{index_autotoc_md3}

\begin{DoxyEnumerate}
\item Pull the image\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{sudo docker pull wpihuron/huron:<tag>}

\end{DoxyCode}
 Currently, {\ttfamily \texorpdfstring{$<$}{<}tag\texorpdfstring{$>$}{>}} can only be a pull request (e.\+g. {\ttfamily pr-\/72}).
\item Run the container in interactive mode\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{sudo docker run -\/it -\/-\/network=host wpihuron/huron:<tag>}

\end{DoxyCode}
 The option {\ttfamily -\/-\/network=host} is needed to expose the network interfaces (including CAN) to the container.
\item To build and run an example code\+:
\end{DoxyEnumerate}

First, {\ttfamily cd} into a sepecific example folder in {\ttfamily examples}, e.\+g. {\ttfamily examples/test\+\_\+robot\+\_\+api}. Each example code is a normal CMake project. To build the code\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{mkdir build \&\& cd build}
\DoxyCodeLine{cmake ..}
\DoxyCodeLine{make}

\end{DoxyCode}


If everything is correct, the binary will be built in {\ttfamily build} folder, which is ready to be executed.

\hypertarget{index_autotoc_md5}{}\doxysection{2. Building from source\+:}\label{index_autotoc_md5}
Clone the main repo and all submodules\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{git clone git@github.com:wpi-\/huron/huron.git -\/-\/recurse-\/submodules}

\end{DoxyCode}
\hypertarget{index_autotoc_md6}{}\doxysubsection{Prerequisites\+:}\label{index_autotoc_md6}

\begin{DoxyEnumerate}
\item ARM toolchains\+: 
\begin{DoxyCode}{0}
\DoxyCodeLine{sudo apt update}
\DoxyCodeLine{sudo apt install gcc-\/aarch64-\/linux-\/gnu g++-\/aarch64-\/linux-\/gnu}

\end{DoxyCode}

\item Build and install third-\/party CAN library 
\begin{DoxyCode}{0}
\DoxyCodeLine{cd third\_party/libsockcanpp}
\DoxyCodeLine{mkdir build}
\DoxyCodeLine{cd build}
\DoxyCodeLine{cmake .. -\/DCMAKE\_TOOLCHAIN\_FILE=../../../tools/<x86\_64 or arm64>-\/toolchain.cmake -\/DBUILD\_SHARED\_LIBS=ON}
\DoxyCodeLine{make}
\DoxyCodeLine{sudo make install}

\end{DoxyCode}

\item Build and install third-\/party Serial library 
\begin{DoxyCode}{0}
\DoxyCodeLine{cd third\_party/serial}
\DoxyCodeLine{mkdir build}
\DoxyCodeLine{cd build}
\DoxyCodeLine{cmake .. -\/DCMAKE\_TOOLCHAIN\_FILE=../../../tools/<x86\_64 or arm64>-\/toolchain.cmake -\/DBUILD\_SHARED\_LIBS=ON}
\DoxyCodeLine{make}
\DoxyCodeLine{sudo make install}

\end{DoxyCode}

\end{DoxyEnumerate}\hypertarget{index_autotoc_md7}{}\doxysubsection{Build and install\+:}\label{index_autotoc_md7}

\begin{DoxyEnumerate}
\item Make sure you are in the root of this repo ({\ttfamily huron/})
\item Create {\ttfamily build} folder 
\begin{DoxyCode}{0}
\DoxyCodeLine{mkdir build}

\end{DoxyCode}

\item Build the project 
\begin{DoxyCode}{0}
\DoxyCodeLine{cd build}
\DoxyCodeLine{cmake .. [-\/DBUILD\_TYPE=<build-\/type>] [-\/DUSE\_PINOCCHIO=1]}
\DoxyCodeLine{make}

\end{DoxyCode}

\item Install {\ttfamily huron} 
\begin{DoxyCode}{0}
\DoxyCodeLine{sudo make install}

\end{DoxyCode}

\end{DoxyEnumerate}

Notes\+:


\begin{DoxyItemize}
\item Currently, the project can be built on Linux only
\item By default, the project builds for Raspberry Pi 64-\/bit (arm64). To change platform, {\ttfamily BUILD\+\_\+\+TYPE} needs to be changed. For example, on Linux x86\+\_\+64\+: {\ttfamily -\/DBUILD\+\_\+\+TYPE=x86\+\_\+64}
\end{DoxyItemize}
